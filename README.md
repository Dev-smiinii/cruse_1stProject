## 2. 커밋 컨벤션
> ### 커밋 유형
> + FEAT : 새로운 기능 추가
> + FIX : 버그 수정, 기능 수정
> + DOCS : 문서 수정

## 3. 코드 컨벤션
> + 단어 단위 한 칸 띄어쓰기 적용
> + 의미 단위 종료 시 ; 적용
> + 변수명 작성 기준 : 수업 내용 & Camel 표기법
> + 예시
   > + let i = 0;
   > + const codeConvention = "띄어쓰기 & Camel 표기법 & 세미콜론(;) 적용";
   > + const a = function (param1, param2) {
   > + }

## 4. Pull requests
> + PR 제목의 형식(예시)
    요청인의 성명과 작업 내용의 요약본을 기재함.
    성명 : 작업 내용 요약
> + PR 본문의 형식(예시)
    Commit message를 복사 후 붙여넣기.
    "fix : 파일명 (수정된 내용을 기입) 수정"
